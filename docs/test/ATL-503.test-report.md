## 테스트 상세

### TC-REG-01: 빈 노드 트리 처리
- 빈 `children` 배열 → `frameCount=0` 정상 반환 ✅
- `children` 누락 (undefined) → nullish coalescing `??` 처리 정상 ✅
- TEXT 없는 FRAME 트리 → `extractTextNodes` 빈 배열 반환 ✅
- 완전히 빈 노드 객체 → 크래시 없이 안전하게 처리 ✅

### TC-REG-02: 중첩 텍스트 노드 추출
- 3단계 중첩 TEXT 노드 → 경로 포함 정확히 추출 ✅
- `characters` 빈 문자열 → falsy 판별로 추출 제외 ✅
- 복수 TEXT 노드 (3개) → 전체 수집 ✅

### TC-REG-03: 잘못된 입력 스키마 검증
- `node_id` 빈 문자열 → 스키마 통과 (형식 제약 없음) ✅
- `node_id` 숫자 타입 → 거부 ✅
- `node_id` 누락 → 거부 ✅
- `rotation` 소수점/음수 → 제약 없어 통과 ✅
- `width` 0/음수 → 허용 (Figma에서 처리) ✅

### TC-REG-04: operations 배열 경계값
- 빈 배열 `[]` → 통과 ✅
- 99개, 100개 → 통과 ✅
- 101개, 1000개 → max(100) 초과로 거부 ✅

### TC-REG-05: Write 스키마 엣지케이스
- `batch_create_nodes`: name 빈 문자열, optional 필드 전부 누락 → 통과 ✅
- `batch_update_auto_layout`: NONE, SPACE_BETWEEN → 통과; padding 부분 누락 → 거부 ✅
- `batch_update_text`: font_size 0, font_name style 빈 문자열 → 통과 ✅
- `batch_update_fills`: 빈 배열(클리어), 복수 fill → 통과 ✅
- `batch_bind_variables`: 복수 bindings (null 포함) → 통과 ✅
- `batch_reorder_move`: MAX_SAFE_INTEGER → 통과; 소수(1.5) → int 제약으로 거부 ✅

### TC-REG-06: 브릿지 미연결 에러 응답 구조
- `content[0].type === "text"` 유지 ✅
- JSON 파싱 가능 ✅
- `error` 필드 string 타입 포함 ✅
- 서버 크래시 없이 배열 반환 ✅

### TC-REG-07: 재연결 상태 시뮬레이션
- 초기 미연결 상태 ✅
- connect() / disconnect() 전환 ✅
- 3회 사이클 후 재연결 정상 ✅
