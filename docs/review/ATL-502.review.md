## Code Quality Findings

### Critical
없음

### Warning

**W-01: `toText()` NaN/Infinity silent data loss** (`src/tools/write.ts:25-27`)
```ts
function toText(result: unknown): string {
  return JSON.stringify(result) ?? '{"error":"Plugin returned undefined result"}';
}
```
- `JSON.stringify(NaN)` → `"null"`, `JSON.stringify(Infinity)` → `"null"` (JS 표준)
- 플러그인이 NaN을 반환하면 에러 없이 `"null"`이 MCP 응답으로 전달됨
- 클라이언트가 실패 케이스를 정상 null 응답으로 해석할 수 있음
- **권고**: `typeof result === "number" && !isFinite(result)` 감지 후 에러 응답 반환

**W-02: `extractTextNodes()` name 누락 시 path 소실** (`src/tools/read.ts:16`)
```ts
const cur = path ? `${path}/${node.name}` : node.name;
```
- `node.name = undefined`(falsy) 인 루트 노드 → `cur = undefined` → 해당 계층 path 소실
- 자식 노드의 path에서 루트 컨텍스트가 완전히 제거됨 (ex: `"Root/Label"` → `"Label"`)
- Figma REST API는 항상 name을 제공하나 방어 코드 없음
- **권고**: `node.name ?? "(unnamed)"` 로 폴백 처리

**W-03: `plugin-bridge.ts` 재연결 시 pending requests 미정리** (`src/plugin-bridge.ts:26-29`)
```ts
this.wss.on("connection", (ws) => {
  this.client = ws; // 기존 pending 정리 없음
```
- 재연결 시 이전 pending 요청들이 30s 타임아웃까지 대기
- 새 클라이언트는 이전 요청 ID를 알 수 없어 응답 불가
- 장시간 UX 저하 (30s timeout 에러)
- **권고**: 재연결 이벤트에서 기존 pending을 즉시 reject 처리

**W-04: `package.json` test:all — CI에서 rest-tools.test.ts 실패 위험** (`package.json:12`)
```json
"test:all": "npx tsx --test src/__tests__/*.test.ts"
```
- `rest-tools.test.ts`는 `docs/test/test-config.json` 없으면 즉시 throw
- CI 환경에서 test:all 실행 시 config 없는 경우 전체 테스트 실패
- **권고**: `test:ci`를 별도 추가해 schema + regression + bug-candidates만 실행

### Info

- `ToolResult` 타입이 정의되어 있으나 `write.ts`는 실제로 `toText()` → `string`만 반환
- 플러그인 응답 구조(`makeEnvelope`)와 MCP 반환 타입 간 정합성 검증 없음

**I-02: `plugin-bridge.ts:78` non-null assertion + race condition**
```ts
this.client!.send(JSON.stringify({ id, type, params }));
```
- `isConnected` 체크 후 send 사이에 연결이 끊기면 예외 발생 (try/catch 없음)
- Promise는 reject 없이 타임아웃까지 대기하게 됨
- **권고**: send를 try/catch로 감싸고 에러 시 즉시 reject
