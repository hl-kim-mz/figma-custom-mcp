# ATL-501 테스트 계획 및 시나리오 작성

**Epic**: ATL-500 — [figma-custom-mcp] QA 통합테스트
**담당자**: 김혜리
**작성일**: 2026-06-23
**상태**: 진행 중

---

## 1. 테스트 목적

figma-custom-mcp MCP 서버(v현재 기준)의 17개 tool에 대해:
- 정상 동작(Happy Path) 검증
- 에러 핸들링(Edge Case) 검증
- 브릿지 연결 상태별 동작 차이 확인

---

## 2. 테스트 환경

| 항목 | 값 |
|------|-----|
| MCP 서버 포트 | 3055 (WS_PORT) |
| 실행 명령 | `node dist/index.js` |
| Figma 연결 | Bridge Plugin (Plugins > Development > figma-custom-mcp) |
| 테스트 Figma 파일 | 테스트용 파일 key 별도 준비 필요 |
| UNSAFE_MODE | false (기본값) |

### 브릿지 상태 조건
- **Connected**: Figma 앱에서 Bridge Plugin 실행 중
- **Disconnected**: Figma 종료 또는 플러그인 미실행

---

## 3. 도구 분류 및 테스트 범위

### 3-A. REST API 기반 Read Tools (5개)
브릿지 연결 불필요, FIGMA_TOKEN + file_key 필요

| # | Tool | 주요 입력 | 기대 출력 |
|---|------|-----------|-----------|
| 1 | `get_page_structure` | file_key | 페이지/프레임 계층 JSON |
| 2 | `get_all_text` | file_key, [page_name], [frame_name] | 텍스트 노드 배열 |
| 3 | `list_components` | file_key | 컴포넌트 목록 |
| 4 | `get_design_tokens` | file_key | 변수/스타일 JSON |
| 5 | `find_node` | file_key, node_name | 노드 ID 목록 |

### 3-B. Bridge Plugin Read Tools (3개)
브릿지 연결 필요

| # | Tool | 주요 입력 | 기대 출력 |
|---|------|-----------|-----------|
| 6 | `get_node_tree` | [depth], [node_id] | 현재 페이지 트리 |
| 7 | `inspect_scope_tree` | file_key, scope_node_id | 스코프 서브트리 정보 |
| 8 | `plugin_status` | (없음) | 연결 상태 JSON |

### 3-C. Bridge Plugin Write Tools (9개)
브릿지 연결 필요, all-or-nothing 배치

| # | Tool | 대상 노드 타입 | 최대 ops |
|---|------|---------------|----------|
| 9 | `batch_create_nodes` | FRAME/TEXT/RECT/ELLIPSE/LINE/COMPONENT | 100 |
| 10 | `batch_create_instances` | 컴포넌트 → 인스턴스 | 100 |
| 11 | `batch_update_geometry` | 모든 노드 | 100 |
| 12 | `batch_update_auto_layout` | FRAME 한정 | 100 |
| 13 | `batch_update_text` | TEXT 한정 | 100 |
| 14 | `batch_update_fills_strokes_effects` | 모든 노드 | 100 |
| 15 | `batch_bind_variables` | 모든 노드 | 100 |
| 16 | `batch_update_component_properties` | INSTANCE 한정 | 100 |
| 17 | `batch_reorder_move` | 모든 노드 | 100 |

## 4. 테스트 시나리오 목록

### TC-R01 ~ TC-R05: REST Read Tools

| TC ID | Tool | 시나리오 | 판정 기준 |
|-------|------|---------|-----------|
| TC-R01 | get_page_structure | 정상 file_key → 전체 페이지 반환 | pages 배열 1개 이상 |
| TC-R02 | get_page_structure | page_name 필터 → 해당 페이지만 반환 | totalPages 일치 |
| TC-R03 | get_page_structure | 30초 캐시 동작 확인 | 2번째 호출 응답 빠름 |
| TC-R04 | get_all_text | 전체 파일 텍스트 추출 | total > 0 |
| TC-R05 | get_all_text | page_name + frame_name 필터 | 범위 내 텍스트만 반환 |
| TC-R06 | list_components | 컴포넌트 있는 파일 | total > 0, name/key/id 존재 |
| TC-R07 | get_design_tokens | 변수 있는 파일 | tokens 컬렉션 반환 |
| TC-R08 | find_node | 존재하는 노드명 검색 | found > 0, id 포함 |
| TC-R09 | find_node | 존재하지 않는 이름 | found === 0 |
| TC-R10 | find_node | limit 파라미터 제한 | results.length <= limit |

### TC-B01 ~ TC-B06: Bridge Tools

| TC ID | Tool | 시나리오 | 판정 기준 |
|-------|------|---------|-----------|
| TC-B01 | plugin_status | 브릿지 연결 상태 | connected: true |
| TC-B02 | plugin_status | 브릿지 미연결 상태 | connected: false + 안내 메시지 |
| TC-B03 | get_node_tree | depth=2, 연결 상태 | 노드 트리 반환 |
| TC-B04 | get_node_tree | 브릿지 미연결 | error 메시지 반환 (크래시 없음) |
| TC-B05 | inspect_scope_tree | 유효한 FRAME scope_node_id | 서브트리 반환 |
| TC-B06 | inspect_scope_tree | 브릿지 미연결 | BRIDGE_NOT_CONNECTED 에러 코드 |

### TC-W01 ~ TC-W09: Write Tools

| TC ID | Tool | 시나리오 | 판정 기준 |
|-------|------|---------|-----------|
| TC-W01 | batch_create_nodes | FRAME 1개 생성 | 성공 응답, Figma에 노드 확인 |
| TC-W02 | batch_create_nodes | TEXT 생성 + characters 지정 | 텍스트 내용 일치 |
| TC-W03 | batch_create_instances | 로컬 컴포넌트 인스턴스 생성 | 인스턴스 ID 반환 |
| TC-W04 | batch_update_geometry | x/y/width/height 변경 | 좌표 일치 |
| TC-W05 | batch_update_auto_layout | HORIZONTAL layout 설정 | layoutMode 변경 확인 |
| TC-W06 | batch_update_text | characters + font_size 변경 | 텍스트/폰트 반영 |
| TC-W07 | batch_update_fills_strokes_effects | SOLID fill 변경 | 색상 반영 |
| TC-W08 | batch_bind_variables | opacity에 변수 바인딩 | 바인딩 확인 |
| TC-W09 | batch_reorder_move | 노드 reparent | 레이어 순서 변경 확인 |

### TC-E01 ~ TC-E05: 에러/엣지케이스

| TC ID | 시나리오 | 판정 기준 |
|-------|---------|-----------|
| TC-E01 | 잘못된 file_key | 에러 메시지 반환, 서버 크래시 없음 |
| TC-E02 | scope_node_id 범위 외 노드 ID | OUT_OF_SCOPE_NODE 에러 |
| TC-E03 | TEXT가 아닌 노드에 batch_update_text | INVALID_NODE_TYPE 에러 |
| TC-E04 | operations 배열 100개 초과 | Zod validation 에러 |
| TC-E05 | MCP 서버 재시작 후 재연결 | 재연결 시 정상 동작 |
